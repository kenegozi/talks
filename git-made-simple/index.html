what is a merge?

moving a branch pointer to a different commit, covering current and merged-with branch history

what is a merge?

$ git man merge


--ff
 When the merge resolves as a fast-forward, only update the branch
 pointer, without creating a merge commit. This is the default
 behavior.

--no-ff
 Create a merge commit even when the merge resolves as a fast-forward.
 This is the default behaviour when merging an annotated (and possibly
 signed) tag.

--ff-only
 Refuse to merge and exit with a non-zero status unless the current
 HEAD is already up-to-date or the merge can be resolved as a
 fast-forward.

more references

also post-it notes
  • branch - moves with commits, merge, rebase, etc.
  • tag - static (e.g. identify a release)
  • HEAD - the current checked-out commit
  • relative - HEAD^^^ is HEAD-3, HEAD~2 is HEAD-2

more references

[somebranch] $ checkout REF
  => HEAD = REF
[current] $ checkout COMMIT_HASH 
  => HEAD = &COMMIT_HASH
[current] $ checkout -b feature 
  => var feature = current
[current] $ reset REF 
  => HEAD = current = REF
[current] $ reset COMMIT 
  => HEAD = current = &COMMIT
